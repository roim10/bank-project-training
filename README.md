# Bank Project

Учебный проект на Go: симулятор банка с конкурентными переводами между счетами. Демонстрирует работу с горутинами, каналами, мьютексами и обработкой ошибок (включая panic/recover) без внешних зависимостей — только стандартная библиотека.

## Что внутри

- **`errors.go`** — именованные ошибки домена (`ErrInsufficientFunds`, `ErrAccountNotFound`, `ErrInvalidAmount`, `ErrAccountAlreadyExists`)
- **`account.go`** — тип `Account` (ID, владелец, баланс, защищённый `sync.Mutex`) с методами `Deposit`, `Withdraw`, `GetBalance`; интерфейс `Transactor`
- **`bank.go`** — тип `BankStorage`, хранилище счетов на `map[string]*Account` за `sync.RWMutex`, с методами `AddAccount`, `GetAccount`, `TotalBalance` и главным методом `Transfer`
- **`main.go`** — точка входа: создаёт 10 счетов по 1000 на балансе, запускает 100 горутин со случайными переводами, собирает результаты через канал и WaitGroup, печатает статистику

## Как это работает

1. При старте создаётся 10 счетов с ID `"0"`–`"9"`, каждый с балансом 1000.
2. Запускается 100 горутин. Каждая выбирает случайные `from`/`to` из существующих счетов и случайную сумму (иногда отрицательную — специально, чтобы проверить обработку паники).
3. Каждая горутина вызывает `BankStorage.Transfer`, который:
   - паникует, если сумма отрицательная (ловится через `defer`/`recover` в этой же функции и превращается обратно в обычную ошибку);
   - находит оба счёта, списывает сумму с отправителя и зачисляет получателю;
   - если списание прошло, а зачисление не удалось — возвращает списанное обратно отправителю (откат), чтобы деньги не терялись.
4. Результат каждого перевода (`from`, `to`, `amount`, `error`) отправляется в канал.
5. Отдельная горутина ждёт завершения всех переводов (`wg.Wait()`) и закрывает канал.
6. Главная горутина читает канал через `range`, считает успешные и неуспешные переводы, выводит сводку и итоговый баланс банка.

## Как запустить

```
go run .
```

Проверка на гонки данных (обязательно перед тем как считать проект завершённым):

```
go run -race .
```

## Пример вывода

```
Успешно: 96, Ошибок: 4
10000
```

Число 10000 — это сумма балансов всех 10 счетов (10 × 1000), она должна оставаться неизменной после любого количества переводов. Если это число меняется — значит где-то в логике перевода или отката есть баг.
